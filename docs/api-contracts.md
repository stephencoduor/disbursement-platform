## 1. API Conventions

### 1.1 Base URL

All endpoints are prefixed with:

```
https://api.disbursepro.co.zm/api/v1
```

### 1.2 Authentication

Every request (except `/auth/login`, `/auth/register`) must include a Bearer JWT issued by Keycloak:

```
Authorization: Bearer <access_token>
```

Access tokens expire after 15 minutes (900 seconds). Use the refresh endpoint to obtain a new token pair.

### 1.3 Required Headers

| Header | Required | Description |
|---|---|---|
| `Authorization` | Yes (except public endpoints) | `Bearer <JWT>` |
| `Content-Type` | Yes (for request bodies) | `application/json` |
| `Accept` | Recommended | `application/json` |
| `X-Idempotency-Key` | Yes (financial POSTs) | UUID v4 -- prevents duplicate processing |
| `X-Request-Id` | Auto-generated by gateway | UUID v4 -- correlation ID for tracing |
| `X-Company-Id` | Injected by gateway | Extracted from JWT `companyId` claim (not user-supplied) |

### 1.4 Pagination (Cursor-Based)

All list endpoints use cursor-based pagination:

**Request Parameters:**

| Param | Type | Default | Max | Description |
|---|---|---|---|---|
| `cursor` | string | - | - | Opaque cursor from previous response |
| `limit` | integer | 20 | 100 | Items per page |

**Response Envelope:**

```typescript
interface PaginatedResponse<T> {
  data: T[];
  cursor: {
    next: string | null;   // Pass as ?cursor= for next page
    prev: string | null;   // Pass as ?cursor= for previous page
  };
  total: number;            // Total matching items
}
```

**Example:**

```json
{
  "data": [ ... ],
  "cursor": {
    "next": "eyJpZCI6IkRTQi0wMTUwIn0=",
    "prev": "eyJpZCI6IkRTQi0wMTAxIn0="
  },
  "total": 284
}
```

### 1.5 Error Format

All errors follow a consistent structure:

```typescript
interface ApiError {
  code: string;             // Machine-readable error code (e.g., "ERR_INSUFFICIENT_BALANCE")
  message: string;          // Human-readable description
  details?: object;         // Additional context (field validation errors, etc.)
  traceId: string;          // X-Request-Id for support debugging
}
```

**Example:**

```json
{
  "error": {
    "code": "ERR_INSUFFICIENT_BALANCE",
    "message": "Insufficient wallet balance. Available: ZMW 1,180.00, Required: ZMW 4,140.00 (gross amount including fees).",
    "details": {
      "field": "netAmount",
      "constraint": "walletBalance",
      "available": 118000000,
      "required": 414000000
    },
    "traceId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
  }
}
```

**Common HTTP Status Codes:**

| Code | Meaning |
|---|---|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request -- validation error |
| 401 | Unauthorized -- missing or expired token |
| 403 | Forbidden -- insufficient permissions |
| 404 | Not Found |
| 409 | Conflict -- duplicate idempotency key or resource conflict |
| 422 | Unprocessable Entity -- business rule violation |
| 429 | Too Many Requests -- rate limit exceeded |
| 500 | Internal Server Error |
| 503 | Service Unavailable -- carrier API down |

**Standard Error Codes:**

| Code | HTTP Status | Description |
|---|---|---|
| `ERR_UNAUTHORIZED` | 401 | Missing or invalid authentication token |
| `ERR_TOKEN_EXPIRED` | 401 | Access token has expired |
| `ERR_FORBIDDEN` | 403 | Insufficient permissions for this action |
| `ERR_NOT_FOUND` | 404 | Resource not found or not in tenant scope |
| `ERR_VALIDATION` | 400 | Input validation failure |
| `ERR_INSUFFICIENT_BALANCE` | 422 | Wallet available balance too low |
| `ERR_LIMIT_EXCEEDED` | 422 | Disbursement exceeds tier limit |
| `ERR_DUPLICATE_DISBURSEMENT` | 409 | Potential duplicate detected |
| `ERR_EMPLOYEE_INACTIVE` | 422 | Employee is not active |
| `ERR_CARRIER_UNAVAILABLE` | 503 | Carrier API is down or degraded |
| `ERR_IDEMPOTENCY_CONFLICT` | 409 | Idempotency key already used with different payload |
| `ERR_RATE_LIMIT` | 429 | Too many requests |
| `ERR_INTERNAL` | 500 | Unexpected server error |

### 1.6 Date Format

All dates use ISO 8601 with Central Africa Time (CAT, UTC+02:00):

```
2026-04-03T14:30:00+02:00
```

### 1.7 Currency & Money

Monetary amounts are always represented as **integers in the smallest unit (ngwee)** to avoid floating-point errors. 1 ZMW = 100 ngwee.

```typescript
interface Money {
  amount: number;       // Integer, smallest unit ngwee (e.g., 150000 = ZMW 1,500.00)
  currency: string;     // ISO 4217 code: "ZMW"
}
```

All `amount` fields in request/response payloads are ngwee integers unless explicitly noted otherwise.

### 1.8 Rate Limiting

| Scope | Limit |
|---|---|
| General (per user) | 60 requests/minute |
| Auth endpoints | 10 requests/minute |
| Financial POST endpoints | 20 requests/minute |
| Platform operator endpoints | 120 requests/minute |

Rate limit headers on every response:

```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 47
X-RateLimit-Reset: 1712160000
```

### 1.9 Idempotency

All disbursement POST endpoints require an `X-Idempotency-Key` header (UUID v4). Keys are cached for 24 hours. Replaying a request with the same key and identical payload returns the original response with HTTP 200. Replaying with the same key but a different payload returns `ERR_IDEMPOTENCY_CONFLICT` (409).

### 1.10 Multi-Tenant Scoping

All company-scoped endpoints automatically filter data by the `companyId` extracted from the JWT. The `X-Company-Id` header is injected by the API gateway -- clients never supply it manually. Platform operator JWTs bypass tenant scoping for cross-company queries.

## 2. Authentication -- `/api/v1/auth`

### 2.1 POST /auth/login

Authenticate a company user or platform operator.

**Request:**

```typescript
interface LoginRequest {
  companyCode?: string;     // Required for company users (e.g., "CPTRAN"), omitted for platform operators
  email: string;            // "mwamba.kapumba@copperbelt-transport.co.zm"
  password: string;         // Min 8 chars, 1 upper, 1 lower, 1 number, 1 special
}
```

**Response: 200 OK**

```typescript
interface LoginResponse {
  data: {
    accessToken: string;      // JWT (RS256, 15-minute TTL)
    refreshToken: string;     // Opaque token (7-day TTL)
    expiresIn: number;        // 900 (seconds)
    tokenType: "Bearer";
    mfaRequired?: boolean;    // true if MFA step needed before full access
    mfaChallenge?: "TOTP" | "SMS";
    user: {
      id: string;             // "USR-001"
      firstName: string;
      lastName: string;
      email: string;
      role: UserRole;
      companyId?: string;     // null for platform operators
      companyName?: string;
    };
  };
}

type UserRole = "company_admin" | "finance_manager" | "approver" | "viewer" | "platform_operator" | "platform_admin";
```

**Example:**

```json
// Request
POST /api/v1/auth/login
Content-Type: application/json

{
  "companyCode": "CPTRAN",
  "email": "mwamba.kapumba@copperbelt-transport.co.zm",
  "password": "S3cure!Pass"
}

// Response: 200 OK
{
  "data": {
    "accessToken": "eyJhbGciOiJSUzI1NiIs...",
    "refreshToken": "dGhpcyBpcyBhIHJlZn...",
    "expiresIn": 900,
    "tokenType": "Bearer",
    "user": {
      "id": "USR-001",
      "firstName": "Mwamba",
      "lastName": "Kapumba",
      "email": "mwamba.kapumba@copperbelt-transport.co.zm",
      "role": "company_admin",
      "companyId": "COMP-001",
      "companyName": "Copperbelt Transport Services Ltd"
    }
  }
}
```

**Error Codes:**

| Code | HTTP | Condition |
|---|---|---|
| `ERR_INVALID_CREDENTIALS` | 401 | Wrong email, password, or company code |
| `ERR_ACCOUNT_LOCKED` | 423 | Account locked after 5 failed attempts |
| `ERR_COMPANY_SUSPENDED` | 403 | Company account is suspended |
| `ERR_MFA_REQUIRED` | 200 | Login successful but MFA verification pending |

## 5. Disbursements -- `/api/v1/disbursements`

> Company Scoped -- automatically filtered by JWT `companyId`.

### 5.1 POST /disbursements

Create a draft disbursement. Draft disbursements do not hold wallet funds until submitted.

**Request:**

```typescript
interface CreateDisbursementRequest {
  employeeId: string;            // "EMP-001"
  netAmount: number;             // ngwee: 150000 = ZMW 1,500.00
  currency: "ZMW";
  purpose: DisbursementPurpose;
  intent: "withdrawal" | "payment";
  costCentre: string;
```

## Appendix C: Disbursement Status Flow

```
draft --> pending_approval --> approved --> processing --> completed
                |                                   |
                v                                   v
            rejected                             failed
                                                    |
                                                    v
                                                 (retry) --> processing
```

## Appendix D: Fee Calculation Formula

```
carrierFee   = max(netAmount * carrierFeePercent / 100, carrierFeeMinimum)
platformFee  = max(netAmount * platformFeePercent / 100, platformFeeMinimum)
levy         = 0  (no current levies in Zambia)
grossAmount  = netAmount + carrierFee + platformFee + levy
```

All intermediate calculations use integer arithmetic in ngwee. No floating-point operations.
